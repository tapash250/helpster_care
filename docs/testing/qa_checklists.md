# QA Checklists

> Reference: `AGENTS.md` §36, §62, §85, §144, Appendix L.

Use these before merging or releasing. Failure of any item blocks the merge /
deployment.

## Authentication
- [ ] Login / logout / session timeout work.
- [ ] Forgot password, reset, email verification.
- [ ] Inactive sessions expire automatically.
- [ ] JWT validated on every privileged request.

## Patient Registration
- [ ] All required fields validated (client + backend).
- [ ] Patient ID generated by backend only, never reused.
- [ ] Timeline event `Registered` created.
- [ ] Audit entry created.
- [ ] Works offline; queued and synced later.

## Hospital Assignment
- [ ] ReBAC scope enforced (`can_access_hospital`).
- [ ] Historical assignments preserved.
- [ ] Bed availability respected.

## Approval Workflow
- [ ] Only authorized roles can transition states.
- [ ] Transitions occur via Edge Functions only.
- [ ] Each transition → timeline + notification + audit.
- [ ] Invalid transitions rejected (`INVALID_WORKFLOW_STATE`).

## Document Upload
- [ ] MIME + size validated; extension not trusted.
- [ ] Stored in private bucket; signed URL issued.
- [ ] Only metadata synced; binary on demand.
- [ ] No PHI in logs.

## Offline Synchronization
- [ ] Critical workflows function offline.
- [ ] Queue survives app restart; auto-retry.
- [ ] Conflicts recorded and surfaced; no silent data loss.
- [ ] Sync status visible in UI.

## Flutter / UI (§144)
- [ ] Material 3, theme tokens, no hardcoded colors.
- [ ] Responsive + accessible (screen readers, ≥48dp, contrast).
- [ ] Loading / Empty / Offline / Error / Permission-Denied / Success states.
- [ ] Widget tests written; reusable components extracted.

## Security (§85)
- [ ] RLS enabled & policies validated.
- [ ] Edge Function authorization tested.
- [ ] Storage secured; signed URLs verified.
- [ ] PHI protected; secrets removed.
- [ ] Hospital & patient assignment enforced.

## Release / Performance / Accessibility
- [ ] Performance budget met (§178).
- [ ] Accessibility validated.
- [ ] Documentation & CHANGELOG updated.
- [ ] Monitoring enabled; rollback documented.
